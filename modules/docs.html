Documentations¶

The detailed descriptions including the architecture are provided in Architecture Description. If you have any questions, please mail to eniacsimon@gmail.com to get help.

Module mouse¶

mouse module contains class Micromouse.

class mouse.Micromouse(mazeMap)¶
The Micromouse represents an agent to perfrom tasks. It maintains:
  • the location in a maze: x, y
  • a local map
  • a task loader
  • the direction it faces: “UP”, “DOWN”, “LEFT”, or “RIGHT”

Before creating a new Micromouse, you may need to create an empty maze map and put this map to create the Micromouse.

addTask(strategy)¶

Add one task to the task list such that task loader will execute the tasks in sequence.

The task is the wrapper of a strategy. You need to put your Strategy as argument.

canGoDown()¶

Return True if there is no wall below your micromouse, and False vice versa.

canGoLeft()¶

Return True if there is no wall on the left of your micromouse, and False vice versa.

canGoRight()¶

Return True if there is no wall on the right of your micromouse, and False vice versa.

canGoUp()¶

Return True if there is no wall above your micromouse, and False vice versa.

goDown()¶

Go Down to the immediate cell of your current location.

As a result, y := y + 1 and direction will be “DOWN”.

goLeft()¶

Go to the Left immediate cell of your current location.

As a result, x := x - 1 and direction will be “LEFT”.

goRight()¶

Go to the Right immediate cell of your current location.

As a result, x := x + 1 and direction will be “RIGHT”.

goUp()¶

Go up to the immediate cell of your current location.

As a result, y := y - 1 and direction will be “UP”.

run()¶

Start to run your tasks by your strategies in the maze.

senseWalls()¶

Use Wall Detector to detector the surrounding walls. If sensors find the walls, the micromouse will update the local map.

You may need to call this function when your micromouse arrives at a cell that has never been visited.

setInitDirection(direction='UP')¶

The initial direction you put micromouse in the maze.

The direction should be “UP”, “DOWN”, “LEFT”, or “RIGHT”. The default direction is “UP”.

setInitPoint(x=-1, y=-1)¶

The initial location (x, y) you put micromouse in the map of a maze.

The coordinate system set the left-top corner as (0, 0):

(0,0) (1,0) (2,0)
(0,1) (1,1) …
(0,2) … …

The x, y will be set failed if it is out of the boundary of maze map. The default location is (-1, -1).

setMotorController(motorController)¶

Specify the MotorController for this micromouse.

Your MotorController should inherit the class MotorController and overrides the function turnLeft(), turnRight(), turnAround(), goStraight().

setSensorController(sensorController)¶

Specify the SensorController for this micromouse.

Your SensorController should inherit the class SensorController and overrides the function senseLeft(), senseRight(), senseUp(), senseDown().

Module map¶

map module contains class Cell and Map.

class map.Map(height, width)¶

The Map class represents a maze map. The single instance of the Map is stored in a micromouse. You can access the local map of a micromouse by mouse.mazeMap.

It maintains:
  • the size of the map: height, width
  • the cells of the map: The cells are represented by a 2-D list array.
getCell(x, y)¶

Get the cell instance by given coordinates x and y.

You may need to first call this function before you update or query a map.

getCellDownWall(cell)¶

Return True if there is a wall below the given cell, otherwise return False.

You may use this function by getCellDownWall(getCell(x, y)) where x and y is the coordinates.

getCellLeftWall(cell)¶

Return True if there is a wall on the left side of the given cell, otherwise return False.

You may use this function by getCellLeftWall(getCell(x, y)) where x and y is the coordinates.

getCellRightWall(cell)¶

Return True if there is a wall on the right side of given cell, otherwise return False.

You may use this function by getCellRightWall(getCell(x, y)) where x and y is the coordinates.

getCellUpWall(cell)¶

Return True if there is a wall above the given cell, otherwise return False.

You may use this function by getCellUpWall(getCell(x, y)) where x and y is the coordinates.

readFromFile(mazeFile)¶

Read and parse the maze file.

The maze file example can be found here. The “|”, “-” or “+” represents the vertical and horizontal walls.

setCellDownAsWall(cell)¶

Update the map that put a wall below the given cell.

You may use this function by setCellDownWall(getCell(x, y)) where x and y is the coordinates.

setCellLeftAsWall(cell)¶

Update the map that put a wall on the left side of the given cell.

You may use this function by setCellLeftWall(getCell(x, y)) where x and y is the coordinates.

setCellRightAsWall(cell)¶

Update the map that put a wall on the right side of the given cell.

You may use this function by setCellRightWall(getCell(x, y)) where x and y is the coordinates.

setCellUpAsWall(cell)¶

Update the map that put a wall above the given cell.

You may use this function by setCellUpWall(getCell(x, y)) where x and y is the coordinates.
